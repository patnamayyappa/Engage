About the Dynamic Ownership Model

The Dynamic Ownership Model (or DOM) is a framework in which records are assigned to the proper user based on set criteria. Generally, this is done shortly after the record is created. It can also occur when a user wants to reevaluate the owner. For instance, inbound interest records owned by a student recruiter need to be assigned to department heads on the date when the academic year begins. This process of assignment or ownership is managed using the following entities:

  • DOM Master – Students in a marketing list are associated with a DOM Master record.
  • DOM Definition Execution Order – This record determines the order of execution of the logic defined in the DOM Definition Logic record.
  • DOM Definition – This record determines the user to who records in the associated DOM Master will be assigned.
  • DOM Definition Logic – This record includes logic based on which records in the associated DOM Master are assigned.
Usage Scenario

Parker Roy, the student recruiter at Acme Student Services, the sub-contractor to Seagull University’s north campus, has completed his student recruitment-related activities. He is winding down tasks as the college’s opening term will begin shortly. Hence, ownership of student records will transition from Parker to Ian Hass, the student coordinator at the north campus. To achieve this assignment, the following steps will be performed:

  1. Create a DOM Master record, Summer Admissions. It includes the Seagull North Campus student group.
  2. Create a DOM Definition record. Ensure that the user Ian Hass is set in this record. This ensures that Ian is assigned student records in the Seagull North Campus student group.
  3. Create a DOM Definition Logic record. It includes logic that will assign records owned by Parker.
  4. Create a DOM Definition Execution Order record. It indicates the order in which logic records, by attribute, are run.
    The operation to assign student records to Ian will occur in the next run of assignment logic that uses values defined in the above records as input parameters. Prior to the next run of the logic, Parker will need to set the DOM Status on all student records that he owns to Pending Assignment.
    Its assumed that other logic is not available that will reassign the records back to Parker Roy.

Note

Assignment logic executes in the background at predefined or custom intervals based on information saved in records of the following entities:

  • DOM Master
  • DOM Definition Execution Order
  • DOM Definition
  • DOM Definition Logic

When account, contact, inbound interest or lifecycle records are assigned, the value in their DOM Status field changes from Pending Assignment to Assignment Complete. Records in which the DOM Status field is set to Locked Assignment are not processed when assignment logic executes.
