About Student Retention

This feature enables users to assign a score to students based on criteria such as performance in a course, assignment; or other criteria such as the student's age, the number of siblings, the number of children, and so on.

For example, to evaluate student performance after completing the East Asian languages course, faculty will create the following records.

  • Score Definition - Enables defining scores for green, yellow and red sections which are broadly equated, from a performance perspective, to Good, Average, and Needs improvement. The following values are set:
    • Green - 80
    • Yellow - 50
    • Red - 35
      The gauge in the Student Success Profile area of the contact will be color-coded based on these definitions.
  • Scoring Factor - Enables creating records that award students one of the above score definition value based on their GPA. The following Scoring Factor records are created:
    • EA_Lang_Good - this record awards a student 80 points if the GPA in the East Asian languages course is in the range 3.2 - 4.
    • EA_Lang_Avg - this record awards a student 50 points if the student's score is in the range 2.4 - 3.2.
    • EA_Lang_NeedsImp - this record awards a student 30 points if the student's score is less than 2.4.
  • Student group - it includes all students enrolled for the East Asian languages course and is associated with the score definition record.

When logic is executed on the student group record at scheduled intervals, each student's record then reflects a score definition based on their GPA in the East Asian languages course.

Administrators can then determine if any remedial action is required to help students improve their scores. This can be done by creating targeted success plans or success network assignment records for students who are awarded a score definition of 50 and 30.
