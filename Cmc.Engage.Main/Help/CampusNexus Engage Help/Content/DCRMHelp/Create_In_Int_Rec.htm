Creating an Inbound Interest Record

As a student advisor, when you create an inbound interest record, ensure that values are specified appropriately in the following fields:

  • Method – lists methods through which the student or student advisor first met each other. For instance, phone call, email, social media, web chat, and so on.
  • Category – indicates the heading under which the initial interaction of the student and the student representative can be categorized.
  • Contact – associate the inbound interest record with a contact record.

Based on requirements, an institution can create additional Method and Category records. They will then be available for selection in the above fields.

Inbound interest records can also be created as a result of integration methods, conducting student seminars, purchasing mailing lists, and so on.

Note

  • If the contact of the inbound interest does not exist in the system, you need to create the associated contact record.
  • Details of the first inbound interest record created for a contact are stored in the Initial Source area of the contact record. The following details are recorded:

    Information in these fields helps an institution perform analytics, such as computing the count of students who modified their initial decisions, the duration after which decisions were modified, and so on.
  • The system automatically creates an inbound interest record when you create a contact record. This behavior is incorporated in the Contact - Create Inbound Interest workflow.
  • When you create an inbound interest, it’s named in the format <Name of contact> - <Value in the Method field> - <date and time of creation>.
