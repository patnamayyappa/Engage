Creating a DOM Master Record

Its required to create a DOM master to select a student group. Records in the group are then assigned to a different user when logic defined in an associated DOM Definition Logic record is executed.

Ensure that values are selected or typed appropriately in the following fields.

  • In the Marketing List field, type text and click to select a value. Alternatively, click to create a new marketing list.
  • In the Run Assignment For Entity field, based on the selected marketing list, one of the following entities is displayed:
    • Account
    • Contact
    • Lifecycle
    • Inbound Interest

    The value that can be selected depends on the entity for which the marketing list is created.

    Note

    The value in this field cannot be changed once the DOM Master has child DOM Definition Execution Order, DOM Definition, or DOM Definition Logic records.

  • In the Attribute Name to be Assigned field, select one of the following:
    • Record Owner – the Owner field of the records in the selected marketing list will be updated with a value based on the condition that is defined.
    • Other User Lookup – The Other User Lookup field displays. An interface displays that assists you to select an alternative field to update.
  • In the Fall Back User field, select a backup user to whom records will be assigned. This behavior occurs in scenarios where the configured assignment does not occur.
