Creating Student Retention Records

To manage the student retention process, you will be required to create records of the following entities:

  • Score Definition
  • Scoring Factor
  • Student Group
Creating a Score Definition Record

When you create a score definition record, you are basically specifying threshold scores in the following fields:

  • Green Score Threshold: The maximum score possible for a student.
  • Yellow Score Threshold: The top level of the middle range score. For instance, if a score from 50 – 80 is considered Good, then this should be set to 50.
  • Red Score Threshold: If a score from 35 – 50 is considered as Needs Improvement, then this should be set to 35.

On saving the record, you will be required to associate scoring factor and student group records. The defined scores will then be assigned to students when assignment logic is executed.

Creating a Scoring Factor Record

Criteria to assign scores defined in the associated score definition record are specified in a scoring factor record.

Ensure that you specify appropriate values in the following fields:

  • Attribute Name - ensure that the specified value is the schema name of the attribute.
  • Points - The points specified here will be awarded to the student if logic defined in the following fields is fulfilled:
  • Condition Type - Select one of the following values
    • Range - When this value is selected, values must be specified in the Min and Max fields.
    • Starts With or Value - A value must be typed in the Value field.

    You can associate multiple scoring factor records with a score definition record. This enables you to create complex criteria where different scores can be assigned to students. The total score will be a combination of all the scoring factors that apply to the student.
