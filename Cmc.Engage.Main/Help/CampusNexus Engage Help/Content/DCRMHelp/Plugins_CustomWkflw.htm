Plugins and Custom Workflows

Enabling the Tracing Service
  1. To enable the plug-in trace log, navigate to the Administrator page and click System Settings.



  2. Navigate to the Customization tab and then select All in the Enable logging to plug-in trace log field.


    When you select:
    • All - all plugin traces will be logged
    • Exception - only logs plugin traces where errors occur

  3. Click OK.
    Note
    Ensure that adequate storage space is available as this operation consumes space rapidly.

Configuring Trace Levels

In addition to the above configuration, the verbosity of the tracing messages can also be controlled. By default, the trace level is set to Error, which adds traces of level Error. You can configure different levels of trace.

This is controlled by an instance of the Configuration entity in which the value of the field ConfigurationName is set to TraceLogLevel.
The Value field defines trace levels. Its possible values are:

  • Trace
  • Debug
  • Info
  • Warn
  • Error
  • Fatal



    Modify this value to change the trace level. After modification, the verbosity of traces entries will change appropriately in the Plug-In Trace log entity.

To check Plug-in Trace Log Entries

  1. View the Plug-in Trace Log grid.
  2. Double click the entry that is created after you updated the object record for which trace is enabled. As trace log records are created asynchronously, it might take a few seconds for the entry to display.


Configuring the Trace Level for an individual Plugin

Trace levels for an individual plug-in can be configured. This is achieved by setting the following JSON in the unsecured configuration of a plug-in:

{

“LogLevel” : “Trace”

}

The unsecured configuration of a plug-in is available while viewing its registration details in the Plug-in Registration Tool.

Using the Dynamics SDK Registration Tool
The tool can be downloaded from this link: https://docs.microsoft.com/en-us/dynamics365/customer-engagement/developer/download-tools-nuget.

In CampusNexus Engage, navigate to the appropriate assembly, locate the Plugin Registration step and open the step details.
