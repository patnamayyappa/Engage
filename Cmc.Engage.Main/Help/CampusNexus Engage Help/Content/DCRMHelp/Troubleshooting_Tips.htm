Troubleshooting Tips

CampusNexus Engage is hosted in an Azure environment. Additionally, all its components are included in the CampusNexus Engage URL.

Following customizations are included:

  1. Plugins
    • Plugins are added to enhance or modify the behavior of CRUD, search and other operations.
    • They are developed as C# libraries
    • These are hosted within CampusNexus Engage using the Registration Utility
      For more information, see https://docs.microsoft.com/en-us/dynamics365/customer-engagement/developer/write-plugin-extend-business-processes.
  2. Custom Workflow Activities
    • Custom activities can be developed and deployed. These can be used in the creation of workflow
    • These are developed as C# libraries
    • These are hosted within CampusNexus Engage using the Registration Utility
      For more information, see https://docs.microsoft.com/en-us/dynamics365/customer-engagement/developer/custom-workflow-activities-workflow-assemblies.
  3. JavaScript Customizations
    • JavaScript code can be added to modify the behavior of
      • Loading of the form
      • Changes to the field values
      • Saving an Entity Instance
    • These are developed using JavaScript and the XRM JavaScript library provided in CampusNexus Engage.
    • These are hosted within CampusNexus Engage as web resources and are loaded on the browser on demand.
    • Custom HTML pages can be hosted within the form or invoked from menus to extend the functionality. These pages using JavaScript and Organization REST APIs can interact with the Dynamics system.
  4. Azure functions
    • Azure functions enable background logic to run at regular intervals
    • These are developed in C#
    • These are hosted within the Azure environment which is external to CampusNexus Engage.
